8. PDF Report Generation Specification
This section details the requirements for generating the user-facing PDF report. The report must be clear, professional, and easy for a non-technical user to understand.

8.1. Generation Logic
Data Source: The report is generated for the current analysis session, not historical data. It will process the list of results obtained after the user clicks the "Analyze Compliance" button.

LLM Response Parsing: A dedicated function will parse the detailed_findings array from the LLM's JSON response. For each violation, it must extract the rule_violated and description.

Recommendation Mapping: This function will also convert the technical description into a simple, actionable recommendation. This can be a rule-based mapping or a separate call to a simpler LLM for rephrasing.

Example:

Input Description: "The text 'METROKIA' is center-aligned. The brand guidelines explicitly forbid center alignment and require strict left-alignment for vertical combinations."

Output Recommendation: "To fix this, align the 'METROKIA' text to the left so its left edge lines up perfectly with the left edge of the Kia logo above it."

8.2. Report Structure
The PDF document will be generated using the fpdf2 library and structured as follows:

Cover Page:

Title: "Brand Compliance Analysis Report"

Date of Generation: The current date and time.

Summary: "This report details the analysis of [X] assets. [Y] assets were found to be non-compliant."

Asset Analysis Pages:

A new page will be started for each asset (image or PDF page) analyzed.

Header: A clear title indicating the source file and page number (e.g., "Analysis for: Campaign_Ad_V2.jpg" or "Analysis for: Brand_Brochure.pdf - Page 3").

Image Display: The analyzed image will be displayed prominently at the top of the page, below the header.

Overall Status: A visually distinct banner showing the final status (e.g., a green "COMPLIANT" or a red "NON-COMPLIANT").

Findings & Recommendations Section:

This section will only appear for non-compliant assets.

It will iterate through each violation found in the LLM response.

For each violation, the report will display:

Violation: A bolded title stating the rule number and name (e.g., "Violation: Rule 2.3. Alignment").

Issue: The technical description of the problem from the LLM's response.

Recommendation: The generated user-friendly, step-by-step advice on how to fix the issue.

A horizontal line will separate each violation for clarity.

Footer:

A footer will appear on every page.

Content: "Report generated by AI Brand Compliance Specialist | Page [current page] of [total pages]"
