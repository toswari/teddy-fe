# Sample Customer Input Document

## Company Overview

**Company:** TechRetail Inc.
**Industry:** E-commerce / Retail
**Size:** 500+ employees
**Annual Revenue:** $50M+

## Current Situation

TechRetail Inc. operates an online retail platform with over 100,000 SKUs across multiple product categories including:

- Electronics
- Home & Garden
- Fashion & Apparel
- Sports & Outdoors

### Current Challenges

1. **Product Search**: Customers struggle to find products using text search alone
2. **Manual Tagging**: Product categorization and tagging is done manually, leading to inconsistencies
3. **Content Moderation**: User-generated content (reviews, photos) requires manual review
4. **Inventory Management**: Visual inspection for quality control is time-consuming

### Technical Environment

- **Platform**: Custom-built on AWS (EKS, RDS, S3)
- **Languages**: Python, JavaScript/TypeScript
- **APIs**: REST-based microservices architecture
- **Current ML**: Basic recommendation engine using collaborative filtering

## Project Goals

1. Implement visual search capabilities (search by image)
2. Automate product categorization and attribute extraction
3. Build automated content moderation for user uploads
4. Explore visual quality inspection for warehouse operations

## Success Criteria

- 30% improvement in search-to-purchase conversion
- 80% reduction in manual tagging effort
- 95% accuracy in content moderation
- Same-day implementation for new product listings

## Timeline

- MVP: 3 months
- Full Production: 6 months
